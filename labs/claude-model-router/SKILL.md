# Claude Model Router — 작업별 모델 추천 & 크레딧 예측

**목적**: 작업을 실행하기 전에 "이 작업엔 어떤 모델이 맞는가"를 판단해서 추천한다. 목표는 두 가지다 — (1) 불필요하게 비싼 모델을 계속 켜놓는 낭비를 막고, (2) 구독 크레딧 범위 안에서 작업 범위를 예측할 수 있게 한다.

**현재 상태**: 가설 기준표 단계. 피카서 님의 실제 이미지/영상 프롬프트 작업 데이터가 아직 쌓이지 않았으므로, 아래 표는 검증 전 추정치다. 매번 사용할 때마다 §4 검증 로그에 실제 결과를 기록하고, 로그가 쌓이면 §1 기준표를 수정한다.

---

## 0. 캐시 비용에 대한 원칙 (먼저 알아야 할 것)

모델 전환 자체를 두려워하지 않는다. 근거:
- 모델을 바꾸면 다음 턴에 전체 대화 히스토리를 캐시 없이 다시 읽는 "1회성 비용"이 발생한다 (예: 4만 토큰 프리픽스 기준 약 $0.20 + 느린 턴 1회).
- 반면 안 맞는 모델을 계속 쓰는 비용은 매 턴 곱해져서 누적된다.
- 결론: **전환 비용 < 잘못된 모델 유지 비용.** 작업 성격이 바뀌는 시점엔 망설이지 말고 전환한다. 다만 한 작업 안에서 왔다갔다 자주 하지 않는다 — "단계가 바뀔 때"에 묶어서 전환.

## 1. 모델 등급별 성격 (가설 — 검증 필요)

| 모델 | 성격 | 컨텍스트/출력 | 비전(이미지 이해) |
|---|---|---|---|
| **Haiku 4.5** | 빠르고 저렴, 단순반복·정형 작업 | 200K / 최대 6.4만 출력 | 지원하나 상대적으로 얕음 |
| **Sonnet 5** | 일상 작업의 기본값, 폭넓게 커버 | 1M / — | 강함 — 대부분의 비전 작업에 충분 |
| **Opus 4.8** | 복잡한 추론이 진짜 필요한 문제 | 1M / 최대 12.8만 출력 | 강함 |
| **Fable 5** | 최상위, 프런티어급 | 1M / 최대 12.8만 출력 | 벤치마크상 비전 태스크에서 Opus보다 우위 (검증 필요) |

가격 구조상 등급이 오를수록 입출력 단가가 배수로 뛴다 (Haiku < Sonnet < Opus < Fable). 등급을 올릴 땐 "정말 필요한가"를 먼저 따진다.

## 2. 작업 유형 기준표 (가설 — 검증 필요)

| 작업 성격 | 권장 모델 | 판단 근거 |
|---|---|---|
| 탐색·근본원인 조사 (원인 불명 오류, 애매한 브리프 좁히기) | Opus 4.8 / Fable 5 | 다단계 추론·장시간 자율 판단이 필요 |
| 아키텍처·파이프라인 설계 결정 | Opus 4.8 | 깊은 사고가 결과 품질을 좌우 |
| 이미지 기반 분석 (레퍼런스 이미지 해석, 캐릭터 아이덴티티 비교) | Fable 5 우선 검토 | 비전 벤치마크 우위 — 실사용 검증 필요 |
| 반복적인 프롬프트 작성 (스토리보드, 샷 리스트, 캐릭터 시트 등 정형 워크플로우) | Sonnet 5 | 대부분의 비전 작업에 충분, 기본값 |
| 코드/스크립트 작성·수정 (Blender bpy 등) | Sonnet 5 → 막히면 Opus 4.8 | 처음부터 고비용 모델 쓸 필요 없음 |
| 번역투 제거, 포맷 정리, 단순 변환 | Haiku 4.5 검토 | 정형화된 반복 작업, 속도·비용 우선 |
| 실험 스킬(-lab류)의 패턴 검증 | Opus 4.8 / Fable 5 | 오류 원인 추적·판단이 잦음 |

### 2-1. 애매할 때의 기본 원칙
- **결과가 곧바로 최종 납품용인가, 탐색/초안용인가?** → 최종: 상위 모델 / 탐색: Sonnet 이하
- **틀리면 되돌리기 비싼 결정인가?** (파이프라인 구조, 캐릭터 아이덴티티 락 설계 등) → Yes: Opus/Fable
- **이미지 해석 정확도가 결과를 좌우하는가?** → Yes: Fable 5 우선 검토
- **작업이 10분 내 끝나는 정형 반복인가?** → Yes: Haiku 검토

## 3. 실행 직전 추천 절차

새로운 작업을 시작하거나 작업 성격이 바뀌는 시점에:

1. 지금 작업이 §2의 어느 행에 해당하는지 판단
2. 애매하면 §2-1 기준으로 좁히기
3. 사용자에게 짧게 안내: "이 작업은 [성격] 특성상 [모델] 추천 — `/model [이름]`으로 전환하시면 됩니다"
4. 전환은 사용자가 직접 판단해서 실행 (자동 전환 아님 — 추천만)
5. 작업 완료 후, 결과가 기대에 맞았는지 확인되면 §4에 기록

## 5. 현재 알려진 한계

- §1, §2의 근거는 코딩·일반 추론 벤치마크에서 가져온 것이며, 이미지/영상 프롬프트 엔지니어링이라는 피카서 님의 실제 도메인에 그대로 맞는지는 검증되지 않았다.
- Fable 5의 "비전 우위"는 벤치마크 수치일 뿐, 실제 캐릭터 시트·톤매칭·레이아웃 분석 작업에서 체감 차이가 있는지는 아직 미확인.
- 이 스킬은 모델 가격·스펙이 바뀌면 §1이 통째로 낡아진다. 큰 모델 개편 소식이 있으면 §1부터 재검토.

## 6. 적용 범위 — 웹 대화창용 스킬과 분리

이 스킬은 **Claude Code(터미널/Desktop)** 전용이다. `/model` 명령어로 같은 세션 안에서 즉시 전환 가능하다는 전제로 만들어졌다.

claude.ai 웹 대화창은 세션 중간에 모델을 못 바꾼다(새 대화창 필요) — 그쪽은 별도의 `web-model-advisor` 스킬이 담당한다. 두 스킬은 판단 기준과 조언 문구가 다르므로 서로의 기준표나 조언 방식을 가져다 쓰지 않는다.
